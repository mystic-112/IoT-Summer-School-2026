# Smart Waste Bin Monitoring System

## Objective

Develop an intelligent IoT-based waste management node capable of monitoring bin fill levels, detecting hazardous conditions, and optimizing municipal waste collection operations.

## Features

* Real-time fill level monitoring
* Waste overflow detection
* Collection priority generation
* Harmful gas detection
* Fire detection
* Bin tampering detection
* Battery monitoring
* MQTT cloud integration
* Dashboard-ready analytics

## Hardware Components

| Component                 | Quantity |
| ------------------------- | -------- |
| ESP32 Development Board   | 1        |
| HC-SR04 Ultrasonic Sensor | 1        |
| MQ135 Gas Sensor          | 1        |
| Flame Sensor              | 1        |
| SW-420 Tilt Sensor        | 1        |
| Active Buzzer             | 1        |
| Status LED                | 1        |

## System Workflow

The HC-SR04 sensor continuously measures the distance between the sensor and the garbage surface to estimate fill percentage.

Additional safety sensors monitor:

* harmful gases generated by decomposing waste
* accidental fires inside bins
* bin displacement or vandalism

The ESP32 processes these readings and generates:

* fill percentage
* collection priority
* estimated pickup schedule
* emergency alerts

The processed data is transmitted to the cloud using MQTT for centralized monitoring by municipal authorities.

## Data Flow

HC-SR04 + MQ135 + Flame Sensor + Tilt Sensor → ESP32 → MQTT → Cloud Platform → Municipal Dashboard

## Collection Priority Logic

| Fill Level | Priority |
| ---------- | -------- |
| 0–49%      | Low      |
| 50–79%     | Medium   |
| 80–94%     | High     |
| 95–100%    | Urgent   |

## Emergency Conditions

An emergency alert is generated if:

* Fill level exceeds 95%
* Harmful gas concentration exceeds threshold
* Fire is detected
* Bin tampering is detected
